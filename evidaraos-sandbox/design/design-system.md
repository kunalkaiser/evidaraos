# EvidaraOS Design System Draft

## Design Goal

Premium enterprise health intelligence. Boardroom-quality, precise, restrained, and credible.

The product should feel like a governed analytical operating system for life-sciences evidence teams.

## Visual Direction

Tone:

- clinical precision
- regulatory seriousness
- executive clarity
- quiet confidence

Avoid:

- generic SaaS cards
- playful gradient blobs
- purple AI aesthetics
- “magic” language
- fake dashboards
- cartoon science imagery

## Color System

Primary palette:

- Deep navy: `#071521`
- Blue black: `#0B1117`
- Ivory: `#F7F1E6`
- Slate: `#5E6B73`
- Mist: `#D8E0E3`
- Steel: `#83919A`

Restrained accents:

- Evidence teal: `#2AA198`
- Clinical blue: `#3A6EA5`
- Signal amber: `#C8943F`
- Validation green: `#3F7D5A`

Use accents sparingly for state, diagrams, and data emphasis.

## Typography

Recommended:

- Display: a serious editorial serif or high-quality grotesk
- Body: readable enterprise sans
- Mono: for IDs, PMIDs, DOIs, audit hashes, schema fields

Do not use oversized hero typography inside dashboards or compact panels.

## Layout Principles

- Dense but calm.
- Strong horizontal sections for public site.
- Workspace should prioritize scanning and repeated action.
- Use tables, timelines, stepper flows, split panes, and status bars.
- Use cards only for repeated items, modals, and contained tools.
- Avoid nested cards.

## Product Visual Motifs

Use architecture visuals based on:

- evidence pipeline
- provenance chain
- protocol-to-report workflow
- validation gate
- module registry
- audit ledger

Good visual primitives:

- node-link evidence maps
- PRISMA-like flows
- provenance columns
- protocol cards
- reviewer queue lanes
- validation checklists
- artifact manifest tables

## Accessibility

- High contrast by default.
- Visible focus states.
- Body text no smaller than 14px in product workspace.
- Do not rely on color alone for validation states.
- Keep motion subtle and nonessential.
